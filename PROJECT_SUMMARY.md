## ğŸ”„ Room Switching Implementation

### Method 1: Normal Switch

**How it works:**
1. User clicks "Normal Switch"
2. `meeting.leave()` is called
3. Current media streams are cleaned up
4. Application state is reset
5. User can join new room with new Room ID

**Code Location:** `src/components/MeetingView.jsx` â†’ `handleNormalSwitch()`

**Characteristics:**
-  Simple and reliable
-  Clean state management
- âŒ Brief disconnection (~1-2 seconds)
- âŒ Requires manual rejoin

### Method 2: Media Relay

**How it works:**
1. User clicks "Media Relay Switch"
2. HLS stream is started in current room
3. Stream URL is generated by VideoSDK
4. Stream can be consumed in target room
5. User maintains connection to original room

**Code Location:** `src/components/MeetingView.jsx` â†’ `handleMediaRelaySwitch()`

**Characteristics:**
-  No disconnection from original room
-  Can broadcast to multiple rooms
- âŒ Requires HLS infrastructure
- âŒ Higher latency (~5-10 seconds)
- âš ï¸ Current implementation is a demonstration

**Production Requirements for Full Media Relay:**
- Backend server for stream management
- RTMP/HLS endpoint configuration
- Stream URL distribution system
- Stream consumption logic in target room

## ğŸ“ Implementation Notes

### Challenges Addressed

1. **Tailwind CSS v4 Compatibility**
   - Issue: New syntax requirements
   - Solution: Updated to use `@import "tailwindcss"` and `@tailwindcss/postcss`

2. **Media Stream Management**
   - Issue: Proper cleanup on room switch
   - Solution: VideoSDK handles cleanup automatically

3. **State Management**
   - Issue: Maintaining state across switches
   - Solution: React state with MeetingProvider context

4. **HLS Latency**
   - Issue: Inherent HLS delay
   - Note: Documented as limitation, RTMP alternative suggested

### Limitations

1. **Media Relay**: Demonstration only - full implementation requires:
   - Backend server for stream management
   - RTMP/HLS infrastructure
   - Stream URL distribution system

2. **Normal Switch**: Brief disconnection period during switch

3. **Browser Support**: Requires modern browser with WebRTC support
